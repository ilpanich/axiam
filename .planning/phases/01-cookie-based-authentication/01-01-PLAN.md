<objective>
Implement cookie-based authentication on the Rust backend: Set-Cookie on login/refresh/MFA-verify, CSRF double-submit middleware, cookie-aware auth extractor, /me endpoint, and cookie-clearing logout.

Purpose: Eliminate XSS vulnerability from sessionStorage token storage by moving JWTs to httpOnly cookies. This is the foundational security change that all other phases depend on.

Output: Backend fully serves auth via cookies with CSRF protection. All existing auth handler tests will need updating (covered in Plan 03).

**4. Create `crates/axiam-api-rest/src/middleware/csrf.rs`:**

Implement an Actix-Web `Transform` + `Service` middleware:

- **Safe methods exempt:** `GET`, `HEAD`, `OPTIONS` pass through without CSRF check.
- **Login/refresh exempt:** Paths ending in `/auth/login`, `/auth/mfa/verify`, `/auth/mfa/setup-confirm`, `/auth/device`, and all `/oauth2/` paths are exempt from CSRF (they don't have a CSRF cookie yet or are token-based). Add a const array `CSRF_EXEMPT_SUFFIXES` for these paths.
- **For state-changing requests (POST/PUT/PATCH/DELETE):**
  - Read `axiam_csrf` cookie value via `req.cookie("axiam_csrf")`
  - Read `X-CSRF-Token` header value via `req.headers().get("X-CSRF-Token")`
  - Compare using `subtle::ConstantTimeEq::ct_eq()` — constant-time comparison per D-01
  - If mismatch or either is missing: return 403 Forbidden using `AxiamError::Forbidden { reason: "CSRF validation failed".into() }`
  - If match: forward request to inner service

- **Cookie helper functions** (put in same file or a sibling `cookies.rs` — Claude's discretion):
  - `pub fn generate_csrf_token() -> String` — uses `rand::Rng::random::<[u8; 32]>()` then `hex::encode()`
  - `pub fn access_cookie(token: &str, max_age_secs: u64) -> Cookie<'static>` — `Cookie::build("axiam_access", token.to_owned()).http_only(true).secure(true).same_site(SameSite::Strict).path("/").max_age(Duration::seconds(max_age_secs as i64)).finish()` (per D-05)
  - `pub fn refresh_cookie(token: &str, max_age_secs: u64) -> Cookie<'static>` — same but `path("/api/v1/auth/refresh")` (per D-06)
  - `pub fn csrf_cookie(token: &str, max_age_secs: u64) -> Cookie<'static>` — `http_only(false)`, `Secure`, `SameSite::Strict`, `path("/")`, max_age same as access token (per D-07, D-09)
  - `pub fn clear_access_cookie() -> Cookie<'static>` — build with empty value, `path("/")`, then `c.make_removal()` (per D-08)
  - `pub fn clear_refresh_cookie() -> Cookie<'static>` — empty value, `path("/api/v1/auth/refresh")`, `make_removal()`
  - `pub fn clear_csrf_cookie() -> Cookie<'static>` — empty value, `path("/")`, `make_removal()`

Use `actix_web::cookie::{Cookie, SameSite, time::Duration}` — re-exported from actix-web, no extra crate needed.

**Important:** Domain attribute is intentionally omitted on all cookies (D-08). `Cookie::build` does not set it by default — correct behavior.

fn cookie_response_from_output(
    out: &LoginOutput,
    config: &AuthConfig,
    user_info: LoginUserInfo,
) -> HttpResponse {
    let csrf_token = generate_csrf_token();

            session_id: out.session_id,

        })
}

**Modify `extract_user()` in `crates/axiam-api-rest/src/extractors/auth.rs` (per D-15):**

After the cached identity check, change token extraction to try cookie first, then fall back to Authorization header (for service-to-service / device auth / backward compat):

```rust
// Try cookie first (browser clients), then Authorization header (service clients)
let token = if let Some(cookie) = req.cookie("axiam_access") {
    cookie.value().to_owned()
} else {
    // Fall back to Authorization: Bearer header for non-browser clients
    let header = req
        .headers()
        .get("Authorization")
        .and_then(|v| v.to_str().ok())
        .ok_or(AxiamError::AuthenticationFailed {
            reason: "missing authentication credentials".into(),
        })?;
    let header = header.trim();
    let mut parts = header.splitn(2, char::is_whitespace);
    let scheme = parts.next().unwrap_or("");
    let credentials = parts.next().unwrap_or("").trim();
    if !scheme.eq_ignore_ascii_case("bearer") || credentials.is_empty() {
        return Err(AxiamError::AuthenticationFailed {
            reason: "invalid Authorization scheme, expected Bearer".into(),
        }.into());
    }
    credentials.to_owned()
};
```

The rest of the function (validate_access_token, building AuthenticatedUser) stays unchanged — it receives a token string regardless of source.
