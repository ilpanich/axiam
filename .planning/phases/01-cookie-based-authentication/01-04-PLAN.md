# Plan 01-04 — Auth path-scope alignment (gap closure)

## Context

Phase 1 verify-work UAT (`01-HUMAN-UAT.md`, 2026-04-15) identified 2 blocker gaps:

1. **`consistent-refresh-path-scope`** — Refresh cookie is issued with `Path=/api/v1/auth/refresh`
   (`csrf.rs:203`), but the handler is registered at `/auth/refresh` (`server.rs:62`,
   `web::scope("/auth")`). Browsers scope cookies by path prefix, so the refresh cookie is never
   sent to the actual handler → silent refresh is unreachable → 401 on token expiry.

2. **`aligned-auth-path-scope-between-client-and-server`** — Admin UI calls
   `/api/v1/auth/login`, but the server serves `/auth/login`. `AuthzMiddleware` treats unknown
   `/api/v1/*` paths as protected, so the request gets 401 before reaching any handler → login
   is impossible from the UI.

Both gaps share one root cause: the auth scope sits at the root path instead of under `/api/v1`.
UAT recommended Option (a) — move the scope — as the smallest blast radius.

## Objective

Move the auth scope from `/auth` to `/api/v1/auth` and propagate the new prefix to every
path constant, OpenAPI annotation, and integration test URI.

## Tasks

### T1 — Move the auth scope

- In `crates/axiam-api-rest/src/server.rs:62`, change `web::scope("/auth")` →
  `web::scope("/api/v1/auth")`.
- Leave the `/oauth2` scope and `/.well-known/openid-configuration` route untouched (out of
  scope; they have well-known URL conventions).

### T2 — Update PUBLIC_PATHS

In `crates/axiam-api-rest/src/permissions.rs`, prefix every `/auth/...` entry in `PUBLIC_PATHS`
with `/api/v1`. Scope of change: lines 181–195 (14 entries):

```
/auth/login                       → /api/v1/auth/login
/auth/register                    → /api/v1/auth/register
/auth/device                      → /api/v1/auth/device
/auth/mfa/verify                  → /api/v1/auth/mfa/verify
/auth/mfa/setup/enroll            → /api/v1/auth/mfa/setup/enroll
/auth/mfa/setup/confirm           → /api/v1/auth/mfa/setup/confirm
/auth/reset                       → /api/v1/auth/reset
/auth/reset/confirm               → /api/v1/auth/reset/confirm

/auth/webauthn/register/start     → /api/v1/auth/webauthn/register/start
/auth/webauthn/register/finish    → /api/v1/auth/webauthn/register/finish
/auth/webauthn/authenticate/start → /api/v1/auth/webauthn/authenticate/start
```

## Out of scope

- `/oauth2/*` routes — not flagged by UAT; have their own OIDC spec-driven URL convention.
- `/.well-known/openid-configuration` — OIDC spec-mandated path.
- Refactoring cookie-construction helpers — the existing `/api/v1/auth/refresh` literal in
  `csrf.rs:203`/`:234` is correct and needs no change.
- Frontend — already calls `/api/v1/auth/*` (see `frontend/src/lib/api.ts:93` for refresh).
