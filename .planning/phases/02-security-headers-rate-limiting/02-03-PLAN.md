    Use `tower-governor` with `SmartIpKeyExtractor` (from `tower_governor::key_extractor`). The pattern from RESEARCH.md:

    ```rust
    //! gRPC rate limiting via tower-governor (per D-10, D-11).

    use std::sync::Arc;
    use tower_governor::{GovernorLayer, GovernorConfigBuilder};
    use tower_governor::key_extractor::SmartIpKeyExtractor;

    /// Build a GovernorLayer for gRPC rate limiting.
    pub fn build_grpc_governor_layer(authz_per_sec: u32) -> GovernorLayer<SmartIpKeyExtractor, governor::state::keyed::DefaultKeyedStateStore<SmartIpKeyExtractor::Key>> {
        let config = GovernorConfigBuilder::default()
            .per_second(1)  // 1 token per second replenishment
            .burst_size(authz_per_sec)
            .key_extractor(SmartIpKeyExtractor)
            .use_headers()
            .finish()
            .expect("valid governor config");

        GovernorLayer {
            config: Arc::new(config),
        }
    }
    ```

    IMPORTANT: Check the exact `tower-governor 0.8.0` API signatures. The `GovernorLayer` struct may need different generic parameters. Check `GovernorConfigBuilder::finish()` return type. The `SmartIpKeyExtractor` reads from `x-forwarded-for` and `x-real-ip` headers automatically.

    If `SmartIpKeyExtractor` is not available in tower-governor 0.8, create a custom `PeerIpKeyExtractor` that reads from `x-forwarded-for` metadata or falls back to connection info.
