  <action>
In `crates/axiam-api-rest/src/server.rs`, split the current `/users` resource (lines 199-203) into two separate resources so that POST can be independently rate-limited:

Current code (lines 199-203):
```rust
.service(
    web::resource("/users")
        .route(web::post().to(handlers::users::create::<C>))
        .route(web::get().to(handlers::users::list::<C>)),
)
```

Replace with:
```rust
.service(
    web::resource("/users")
        .route(web::get().to(handlers::users::list::<C>)),
)
.service(
    web::resource("/users")
        .wrap(build_governor(rate_limit_cfg.register_per_min))
        .route(web::post().to(handlers::users::create::<C>)),
)
```

IMPORTANT: Actix-Web does NOT allow two `web::resource` calls with the same path. Instead, keep the single `/users` resource but apply the rate limiter to the entire resource. The GET (list) will also be rate-limited at 5/min which is acceptable since admin list queries won't hit 5/min in normal use. The alternative approach:

```rust
.service(
    web::resource("/users")
        .wrap(build_governor(rate_limit_cfg.register_per_min))
        .route(web::post().to(handlers::users::create::<C>))
        .route(web::get().to(handlers::users::list::<C>)),
)
```

This follows the exact same pattern used for `/login` (line 65-67), `/reset` (line 118-120), and `/token` (line 139-141).
