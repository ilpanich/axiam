    **IMPORTANT (per D-07 — UPSERT):** Use raw SurrealQL UPSERT via `db.query()` — do NOT use list()+create() strategy. The seeder must be idempotent under concurrent startup. For each `(action, description)` in the registry, execute:

    ```sql
    UPSERT type::record('permissions', $id) SET
      tenant_id = $tenant_id,
      action = $action,
      description = $description,
      created_at = time::now(),
      updated_at = time::now()
    ```

    Generate a deterministic UUID for each permission using a namespace-based approach (e.g., `Uuid::new_v5(&tenant_id_as_namespace, action.as_bytes())`) so the same tenant+action always produces the same record ID. This ensures true idempotency — the UPSERT hits the same record on every restart.

    Alternatively, if SurrealDB v3 does not support UPSERT with `type::record()`, use `INSERT INTO permissions { id: $id, ... } ON DUPLICATE KEY UPDATE updated_at = time::now()`. Test which SurrealQL syntax works and use that.

    Do NOT use `PermissionRepository::list()` then `create()` — this is race-prone under concurrent startup (per RESEARCH.md "Don't Hand-Roll" table).

    **1. Create `crates/axiam-api-rest/src/middleware/authz.rs`** (per D-01, D-03):

    Follow the `CsrfMiddleware` Transform/Service pattern exactly. Implement:
    - `pub struct AuthzMiddleware;` with `Transform<S, ServiceRequest>` impl
    - `pub struct AuthzMiddlewareService<S> { inner: S }` with `Service<ServiceRequest>` impl

    In `call()`:
    - Extract path from `req.path()`.
    - Check `is_public_path(path)` using `PUBLIC_PATHS` from the permissions module. Path matching: iterate PUBLIC_PATHS, for entries ending with `*`, check `starts_with(entry_without_star)`. For entries containing `*` in the middle (like `/federation/*/oidc/callback`), split on `*` and check prefix + suffix. For exact entries, compare directly.
    - If public: forward request unchanged (`fut.await?.map_into_left_body()`).
    - If not public: check for JWT presence — `req.cookie("axiam_access").is_some() || req.headers().contains_key("Authorization")`. If neither present, return 401 using `AxiamApiError(AxiamError::AuthenticationFailed { reason: "authentication required".into() })` and `req.error_response(error).map_into_right_body()`.
    - If JWT present: forward to handler. The per-handler `RequirePermission` does the actual permission check (D-02).

    **2. Add `pub mod authz;` to `crates/axiam-api-rest/src/middleware/mod.rs`.**

    **3. Extend `crates/axiam-api-rest/src/authz.rs`**:

    Add a marker struct `pub struct AuthzChecked;` that handlers insert into request extensions after successful permission check. Add to `RequirePermission::check()` — but since `check()` does not have access to `HttpRequest`, instead document that handlers should call `req.extensions_mut().insert(AuthzChecked)` after `check()`. For defense-in-depth, this is optional in this plan and enforced by the integration test in Plan 05.

    **4. Wire AuthzMiddleware into `crates/axiam-api-rest/src/server.rs`**:

    In `register_api_v1_routes()`, add `.wrap(AuthzMiddleware)` to the `/api/v1` scope. The middleware wraps INSIDE the CSRF middleware (CSRF is on `/auth` scope, AuthzMiddleware is on the outer API v1 scope). The middleware must be added to the `/api/v1/` scope wrapper in server.rs.

    Also add AuthzMiddleware to the `/auth` scope (which is separate from `/api/v1/`). The `/auth` scope already has `CsrfMiddleware` — add AuthzMiddleware inside it.

    Also add AuthzMiddleware to the `/oauth2` scope.

    **5. Wire AuthzChecker + permission seeding into `crates/axiam-server/src/main.rs`**:

    After repository creation and migration run:
    - Create the REST-facing `AuthzChecker`:
      ```rust
      let rest_authz: Arc<dyn axiam_api_rest::authz::AuthzChecker> = Arc::new(
          axiam_authz::AuthorizationEngine::new(
              role_repo.clone(), permission_repo.clone(),
              resource_repo.clone(), scope_repo.clone(), group_repo.clone(),
          )
      );
      ```
    - Add `web::Data::new(rest_authz.clone())` as app_data in the `HttpServer::new` closure.
    - After migrations, call `seed_permissions()` for all existing tenants. Query all tenants via `tenant_repo.list()` with a large pagination, iterate each tenant, call `axiam_db::seed_permissions(&db, tenant.id, axiam_api_rest::permissions::PERMISSION_REGISTRY).await`.
    - Log: `info!("Seeded permissions for {} tenants", count)`.
