<objective>
Wire RequirePermission authorization checks into every non-public handler and implement self-service ownership checks for user-facing endpoints. Also wire seed_permissions into tenant creation so new tenants get permissions auto-seeded.

Purpose: After the AuthzMiddleware (Plan 01) blocks unauthenticated requests with 401, this plan adds the per-handler permission checks that return 403 for unauthorized users. Combined, this achieves full RBAC enforcement.

Output: Every handler calls RequirePermission for its corresponding permission. Self-service endpoints short-circuit with caller_user_id == target_user_id check. New tenants get permissions seeded on creation.
</objective>

    Add `authz: AuthzData` parameter to every handler function signature. Before each handler's business logic, add the permission check. Use `Uuid::nil()` as resource_id for all endpoint-level checks (global roles apply, per Research Pitfall 1).

    - `list`: `RequirePermission::new("users:list", Uuid::nil()).check(&user, authz.get_ref().as_ref()).await?;` — This is the admin user listing endpoint (T19.3). Only users with `users:list` permission can list all users.
    - `get`: Self-service check first — if `is_own_resource(&user, target_id)`, skip authz. Otherwise `RequirePermission::new("users:get", Uuid::nil()).check(...)`.
    - `create`: `RequirePermission::new("users:create", Uuid::nil()).check(...)`.
    - `update`: Self-service check — if `is_own_resource(&user, target_id)`, skip authz (per D-14, D-15). Otherwise `RequirePermission::new("users:update", Uuid::nil()).check(...)`.
    - `delete`: `RequirePermission::new("users:delete", Uuid::nil()).check(...)`. No self-service (users cannot delete themselves).
    - `unlock`: `RequirePermission::new("users:admin", Uuid::nil()).check(...)` (per D-06).

    **3. Wire authz into `handlers/auth.rs`**:

    Most auth endpoints are public (login, register, refresh, device, etc. — in PUBLIC_PATHS). For the non-public ones:
    - `me`: No authz needed — the AuthenticatedUser extractor already ensures authentication. The endpoint returns caller's own data.
    - `logout`: No additional authz — only affects the caller's own session.
    - MFA endpoints (`enroll_mfa`, `confirm_mfa`, `verify_mfa`): Self-service only — these operate on the caller's own account. No permission check needed since AuthenticatedUser extractor validates identity.
    - `setup_enroll_mfa`, `setup_confirm_mfa`: These are CSRF-exempt and use setup_token auth — they should be in PUBLIC_PATHS.
    - `reset_mfa` (at `/users/{id}/reset-mfa`): Self-service check — if own user, allow. Otherwise `RequirePermission::new("users:admin", Uuid::nil()).check(...)`. This is the admin MFA management endpoint (T19.3) — admins with `users:admin` can reset MFA for other users.

    **4. Wire authz into `handlers/mfa_methods.rs`**:

    - `list_mfa_methods`: Self-service check — if `is_own_resource(&user, target_user_id)`, allow. Otherwise `RequirePermission::new("users:admin", Uuid::nil()).check(...)`. Admins can list MFA methods for other users (T19.3).
    - `delete_mfa_method`: Self-service check — if `is_own_resource(&user, target_user_id)`, allow. Otherwise `RequirePermission::new("users:admin", Uuid::nil()).check(...)`. Admins can delete MFA methods for other users (T19.3).
