    **Define return type:**
    ```rust
    pub struct SeedRolesResult {
        pub super_admin_role_id: Uuid,
        pub admin_role_id: Uuid,
        pub viewer_role_id: Uuid,
    }
    ```

    **Implementation** (per D-12):

    1. Create a `SurrealRoleRepository` and `SurrealPermissionRepository` from the db connection.
    2. Check if roles named "super-admin", "admin", "viewer" already exist for this tenant (list + filter by name). If all three exist, return their IDs without creating duplicates.
    3. For each missing role, create via `RoleRepository::create()`:
       - **super-admin**: `CreateRole { name: "super-admin", description: "Full system access — all permissions", is_global: true, ... }`
       - **admin**: `CreateRole { name: "admin", description: "Administrative access — all entity CRUD operations", is_global: true, ... }`
       - **viewer**: `CreateRole { name: "viewer", description: "Read-only access — list and get operations", is_global: true, ... }`
    4. List all permissions for the tenant via `PermissionRepository::list()` with large pagination.
    5. Grant permissions to roles:
       - **super-admin**: ALL permissions (grant every permission in the DB to this role).
       - **admin**: All permissions EXCEPT `admin:bootstrap` (filter out the bootstrap permission).
       - **viewer**: Only permissions whose action ends with `:list` or `:get` (filter by suffix).
    6. Grant permissions via `PermissionRepository::grant_to_role()` for each permission-role pair.
    7. Return `SeedRolesResult` with the three role UUIDs.

    Update `crates/axiam-db/src/lib.rs` to also export `seed_default_roles` and `SeedRolesResult`.
