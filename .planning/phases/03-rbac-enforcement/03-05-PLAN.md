  <action>
    **1. Create `crates/axiam-api-rest/tests/rbac_test.rs`:**

    Follow the existing test pattern from `middleware_test.rs` and other test files. Use `actix_web::test` with in-memory SurrealDB.

    **Setup helper**: Create a `setup_with_authz()` function that:
    - Creates in-memory SurrealDB connection
    - Runs migrations
    - Seeds permissions via `seed_permissions()`
    - Seeds default roles via `seed_default_roles()`
    - Creates a test user and assigns them a specific role
    - Creates the `AuthzChecker` and adds it as app_data
    - Returns the test `App` configured with `register_api_v1_routes`
    - Also returns a JWT minting function that creates valid tokens for test users

    **Test cases** (per Research Validation Architecture):

    - `unauthenticated_returns_401`: Send GET to `/api/v1/users` without any cookie or Authorization header. Assert status 401.
    - `no_permission_returns_403`: Create a user with the "viewer" role. Mint a JWT. Send POST to `/api/v1/users` (which requires `users:create`). Assert status 403.
    - `admin_can_access`: Create a user with the "super-admin" role. Mint a JWT. Send GET to `/api/v1/users`. Assert status 200.
    - `self_service_owner_allowed`: Create a user with the "viewer" role (no `users:update` permission). Mint a JWT. Send GET to `/api/v1/users/{own_id}`. Assert status 200.
    - `self_service_nonowner_denied`: Create a user with the "viewer" role. Mint a JWT. Send PUT to `/api/v1/users/{other_user_id}` with update body. Assert status 403.
    - `public_routes_no_auth_required`: Send GET to `/health`. Assert status 200 (no auth needed). Send POST to `/auth/login` with invalid body. Assert status is NOT 401 (it should be 400 or similar, not an auth error).
    - `all_routes_have_permission` (per D-08): Import `ROUTE_PERMISSION_MAP` and `PERMISSION_REGISTRY`. For each `(method, path, permission)` in `ROUTE_PERMISSION_MAP`, assert that `permission` exists in `PERMISSION_REGISTRY` (i.e., `PERMISSION_REGISTRY.iter().any(|(action, _)| *action == permission)`). This is a compile-time static analysis test — no HTTP calls needed.

    **2. Create `crates/axiam-api-rest/tests/bootstrap_test.rs`:**

    **Test cases:**

    - `bootstrap_creates_admin`: Call POST `/api/v1/admin/bootstrap` with valid `{ org_id, tenant_id, email, username, password }`. Assert status 201. Assert response contains `user_id`.
    - `bootstrap_returns_404_after_admin`: Call bootstrap once (201), then call again with different email. Assert status 404.
    - `bootstrap_rejects_wrong_email`: Set `AXIAM_BOOTSTRAP_ADMIN_EMAIL` env var. Call bootstrap with a non-matching email. Assert status 403. (Use `std::env::set_var` in test — must be single-threaded or use a test-specific approach.)
    - `bootstrap_admin_can_login`: Call bootstrap to create admin. Then call POST `/auth/login` with the bootstrap credentials. Assert login succeeds (status 200 with Set-Cookie).
  </action>
