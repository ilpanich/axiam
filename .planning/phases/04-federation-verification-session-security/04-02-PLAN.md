    Constants: `pub const TTL: Duration = Duration::from_secs(3600);` (D-01), `pub const STALE_WINDOW: Duration = Duration::from_secs(24 * 3600);` (D-03), `pub const FORCED_REFETCH_COOLDOWN: Duration = Duration::from_secs(60);` (D-02).

    Public methods:
    - `pub async fn get_or_fetch(&self, http: &reqwest::Client, key: (Uuid, Uuid), jwks_uri: &str) -> Result<JwkSet, FederationError>` — implements the 3-step flow at RESEARCH §1 "Lookup flow":
        1. Read lock → if entry exists and `fetched_at + TTL > now` → return cloned keys.
        2. Drop read lock → call `fetch_jwks(http, jwks_uri)`. On Ok: write lock; insert/update entry with `fetched_at = now`; return.
        3. On fetch Err: read lock; if entry exists and `fetched_at + STALE_WINDOW > now` → `tracing::warn!(jwks_uri, "serving stale JWKS while IdP unreachable")` and return cached keys. Else propagate the error as `FederationError::JwksFetchFailed`.
    - `pub async fn force_refetch_if_allowed(&self, http: &reqwest::Client, key: (Uuid, Uuid), jwks_uri: &str) -> Result<JwkSet, FederationError>` — called by the verifier when `kid` is unknown. Read lock to inspect `last_refetch_attempt`. If `Some(t)` and `t + FORCED_REFETCH_COOLDOWN > now` → return `Err(FederationError::JwksKidUnknown)` immediately (rate-limited). Otherwise update `last_refetch_attempt = Some(now)` and perform a fetch; on success update keys+fetched_at.

    Step 2 — Implement private helper `async fn fetch_jwks(http: &reqwest::Client, jwks_uri: &str) -> Result<JwkSet, FederationError>` using `http.get(jwks_uri).send().await?.error_for_status()?.json::<JwkSet>().await?`.
