<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → export/delete endpoint | only owner or privileged admin may act |
| export file at rest → disk/DB | must be encrypted; deleted on download/expiry |
| purge job → audit_log | the only sanctioned UPDATE path |
| cancel link → public endpoint | unauthenticated but single-use token-gated |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-5-export-token | Spoofing | export download link | mitigate | Single-use, 24h TTL, server-generated opaque token stored as SHA-256 hash; file deleted on download/expiry (D-13) |
| T-5-export-rest | Information Disclosure | export file at rest | mitigate | AES-256-GCM via encrypt_separate (export_encryption_key); never written plaintext (D-12) |
| T-5-pseudonym | Information Disclosure | audit pseudonym | mitigate | Keyed HMAC-SHA256 pepper (D-02); 64-bit correlation token, not reversible without pepper |
| T-5-delete | Elevation of Privilege | account/delete authorization | mitigate | Ownership check (self) OR `users:erase` permission (D-07); tenant-scoped purge queries |
| T-5-cancel | Elevation of Privilege | public cancel endpoint | mitigate | Single-use hash-stored token; consumed on click; window-bounded; only clears deletion flags |
| T-5-purge-tenant | Elevation of Privilege | purge sweep cross-tenant | mitigate | Every purge/anonymize/pseudonymize query scoped by tenant_id |
| T-5-audit-update | Tampering | audit_log integrity | mitigate | Only pseudonymize_actor UPDATEs (D-04); event facts (action/outcome/timestamp) immutable; FOR delete NONE |
| T-5-SC | Tampering | cargo installs | accept | No new packages — serde_json/aes-gcm/hmac workspace deps |
</threat_model>
