<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| built image → container registry | image is published to GHCR and signed; consumers trust the signature |
| OS base + app layers → runtime | vulnerable OS/library packages in the image become runtime risk |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-05 | Tampering | published container image | mitigate | trivy image scan blocks HIGH/CRITICAL (fixable) before push + cosign sign (D-06) |
| T-06-06 | Spoofing | image signature ↔ scanned content | mitigate | cosign signs the exact pushed digest; cache-from gha guarantees scanned==pushed digest (Pitfall 2) |
| T-06-07 | Information Disclosure | scan findings visibility | mitigate | image-scan SARIF uploaded to GitHub Security tab (D-07) |
| T-06-08 | Elevation of Privilege | unfixable OS-base CVEs | accept | `--ignore-unfixed` skips CVEs with no upstream fix (D-06 explicit); distroless base (Plan 06-03) minimizes surface |
</threat_model>
