<objective>
Close the plaintext-federation-secret defect (SEC-045/SEC-017): `OidcFederationService::handle_callback()` (oidc.rs:292) reads `&config.client_secret` (legacy plaintext) without calling `decrypt_client_secret_or_legacy`, the service has no `encryption_key` field, create/update store the secret in plaintext, and `client_secret` is serialized in REST responses. Wire decrypt-at-use, encrypt-on-write, never-serialize, and prove OIDC login survives a restart.

Purpose: Federation client secrets must be encrypted at rest, decrypted only at use, and never leak through API responses.
Output: Hardened `oidc.rs`, `federation.rs` model, federation REST handler, `main.rs` wiring, and a round-trip + post-restart test.
</objective>

<interfaces>
Existing crypto infra (VERIFIED):
- `axiam_federation::secrets::decrypt_client_secret_or_legacy(key: &[u8;32], nonce: Option<&str>, ciphertext: Option<&str>, legacy_plaintext: &str) -> Result<String, _>` (secrets.rs:75-91) — resolves encrypted columns or falls back to legacy plaintext.
- `axiam_federation::secrets::encrypt_client_secret(key, plaintext) -> (nonce, ciphertext)` — for encrypt-on-write.
- DB repo already has `set_encrypted_secret()` and `list_with_legacy_plaintext_secret()` (federation_config.rs).
- main.rs:122-136,203-216 already loads `AXIAM__AUTH__FEDERATION_ENCRYPTION_KEY` and runs `migrate_plaintext_federation_secrets` boot backfill. The key is NOT currently injected into OidcFederationService.

The gap (oidc.rs:292): replace `&config.client_secret` with
    &axiam_federation::secrets::decrypt_client_secret_or_legacy(
        &self.encryption_key,
        config.client_secret_nonce.as_deref(),
        config.client_secret_ciphertext.as_deref(),
        &config.client_secret,
    ).map_err(|_| FederationError::ConfigIncomplete)?

Steps:
1. Add `encryption_key: [u8; 32]` to OidcFederationService + its `new()`.
2. main.rs: pass `federation_encryption_key` when constructing OidcFederationService.
3. oidc.rs handle_callback: decrypt-at-use (above).
4. REST create/update handler: call `encrypt_client_secret` then `set_encrypted_secret` (move encrypt OUT of the DB layer — the DB create has a TODO(T19.8); plaintext must never reach `CREATE ... client_secret = $client_secret`).
5. Never-serialize: add `#[serde(skip)]`/`skip_serializing` to `client_secret`, `client_secret_ciphertext`, `client_secret_nonce`, `client_secret_key_version` in the RESPONSE path. Prefer a `FederationConfigResponse` DTO over skipping on the domain model if skipping breaks stored-data round-trips (Pitfall 4 / A3).

Pitfall 4: `#[serde(skip)]` on the domain `client_secret` will break any test that serializes a FederationConfig and checks the JSON — use a response DTO OR `skip_serializing` (keep deserializing).
Runtime risk: `AXIAM__AUTH__FEDERATION_ENCRYPTION_KEY` MUST be set or the boot backfill is skipped and decrypt fails for backfilled rows. Verify it is set in dev compose + k8s secrets before deploy (operator note in SUMMARY).
NOTE: 3 pre-existing SAML federation_test failures under --no-default-features are a known baseline — a 4th+ is real.
