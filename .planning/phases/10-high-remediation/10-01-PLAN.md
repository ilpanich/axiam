<objective>
Collapse the two password-hashing code paths into one and wire the auth pepper end-to-end so REST-created users authenticate correctly. Closes REQ-14 AC-1 (CQ-B09 / CQ-B01).

Purpose: A duplicated Argon2 hasher in the repo layer (user.rs:152-173) ignores the configured pepper, so any user created through the REST API is hashed WITHOUT the pepper while the login path verifies WITH it — these users can never log in once a pepper is configured. This is both a correctness defect and a single-source-of-truth security defect.

Output: Repo-layer hasher deleted; all hashing routed through axiam-auth::password::hash_password; AXIAM__AUTH__PEPPER loaded at startup and threaded into the user repository; an integration test proving the round-trip.
</objective>
