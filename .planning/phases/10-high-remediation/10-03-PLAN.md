<objective>
Make CPU-bound crypto async-safe and enforce tenant isolation on graph mutations. Closes REQ-14 AC-2 (CQ-B02), AC-4 (CQ-B07 + CQ-B08).

Purpose: (1) Argon2id and PKI keygen/sign run inline on the Tokio executor — a burst of logins or CA generations starves all worker threads (DoS). (2) role/permission edge mutations ignore `_tenant_id`, so a caller can attach a role from tenant B to a user in tenant A (cross-tenant privilege escalation). (3) resource hierarchy can form cycles, orphan children on delete, and silently truncates ancestor walks at depth 50.

Output: spawn_blocking + bounding semaphore around all heavy crypto; tenant-membership-checked, transactional edge mutations; cycle/orphan-safe resource hierarchy that errors on depth overflow.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client→auth (login) | High-frequency untrusted input triggers Argon2 |
| caller→DB edge mutation | tenant_id from request must be enforced, not trusted blindly |
| caller→resource hierarchy | parent_id changes can form cycles / orphan children |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-04 | Denial of Service | axiam-auth Argon2, axiam-pki keygen/sign | mitigate | Move all CPU-bound crypto into `spawn_blocking` behind a shared `Arc<Semaphore>(4)` so a burst cannot saturate Tokio worker threads (CQ-B02) |
| T-10-05 | Elevation of Privilege | axiam-db role.rs / permission.rs edge mutations | mitigate | SurrealQL LET+THROW verifies both endpoints belong to the request tenant before RELATE; map THROW to AuthorizationDenied; wrap delete+side-effects in a transaction (CQ-B07/SEC-007) |
| T-10-06 | Tampering / Denial of Service | axiam-db resource hierarchy | mitigate | Reject parent changes that create a cycle; block delete of resources with children; surface depth-overflow as an error instead of silently truncating at 50 (CQ-B08) |
</threat_model>
