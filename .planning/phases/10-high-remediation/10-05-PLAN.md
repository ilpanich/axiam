<objective>
Protocol + input hardening: pagination clamp, generic 5xx, TOTP replay rejection, SAML protocol checks, and federation API completeness. Closes REQ-14 AC-5 (protocol slice): SEC-010/CQ-B30, SEC-011/SEC-039/CQ-B33, SEC-008, SEC-005, CQ-B40.

Purpose: (1) Pagination.limit is unbounded — `limit=999999` enables resource exhaustion. (2) 5xx bodies echo internal errors (DB strings, crypto detail) to clients. (3) TOTP codes can be replayed within their 30s window. (4) SAML responses are not checked for InResponseTo/Destination, Conditions is optional, and SP metadata advertises unsigned assertions (XSW exposure). (5) federation configs can't be fully created via the API (missing idp_signing_cert_pem + algorithms; brittle PEM parser).

Output: clamped pagination, generic 5xx with logged detail, TOTP replay prevention via a last-used step, SAML protocol checks (CI SAML-ON only), and a complete federation create/update API.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client→paginated endpoints | limit is attacker-controlled |
| server→client (errors) | error bodies must not leak internals |
| IdP→SP (SAML/TOTP) | tokens/codes must not be replayable or forgeable |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-11 | Denial of Service | axiam-core Pagination | mitigate | Clamp limit to [1,200] at deserialization so a single request can't fetch unbounded rows (SEC-010/CQ-B30) |
| T-10-12 | Information Disclosure | axiam-api-rest error.rs | mitigate | Generic 5xx message; log internal detail via tracing — stops DB/crypto internals leaking to clients (SEC-011/SEC-039/CQ-B33) |
| T-10-13 | Repudiation / Spoofing | axiam-auth TOTP | mitigate | Persist last-used step; reject step ≤ stored — prevents code replay within a window (SEC-008) |
| T-10-14 | Spoofing | axiam-federation SAML | mitigate | Validate InResponseTo/Destination, require Conditions, sign assertions/requests, bind verified signature ref to consumed assertion (XSW) — CI SAML-ON path only (SEC-005) |
| T-10-15 | Tampering | federation config API | mitigate | Accept + validate idp_signing_cert_pem via the pem crate before persist so SAML signature verification uses an operator-pinned cert (CQ-B40) |
</threat_model>
