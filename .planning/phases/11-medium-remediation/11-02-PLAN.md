<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| server → external webhook URL | Outbound request to attacker-controllable URL (SSRF) |
| IoT device → mTLS endpoint | Client cert presented; must chain to trusted CA |
| public client → OAuth2 authorize | Auth-code interception risk without PKCE |
| client → rate-limited endpoint behind proxy | Spoofed X-Forwarded-For can evade per-IP limits |
| publisher → AMQP broker → consumer | Message body must be authenticated end-to-end |
| federation client → IdP JWKS endpoint | Outbound discovery fetch (SSRF + unbounded body) |
