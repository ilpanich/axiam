<objective>
Make `DbManager` reconnect-safe so an idle WebSocket reconnect can never silently
drop the `use_ns`/`use_db` selection and route queries to the empty default
`main`/`main` namespace (the confirmed Phase-12 root cause, per 13-RESEARCH).

Purpose: Eliminate the silent stale-connection failure mode (REQ-17). SurrealDB
SDK 3.1.3 does NOT replay ns/db on auto-reconnect (open issue #5750, no SDK hook).
The durable fix is a background guard task that re-asserts ns/db + a health_check
that verifies the active selection.

Output: Updated `connection.rs` (guard task + asserting health_check), a new
`DbError::SessionMismatch` variant, and a reproducing regression test.
</objective>
