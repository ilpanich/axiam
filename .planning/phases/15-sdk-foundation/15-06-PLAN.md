  <what-built>Plan 05 scaffolded the SDK package manifests with the locked identities (D-11/D-12). Before any publish pipeline is built (Phases 16-22), the human must confirm these names are available/reserved. This cannot be auto-reserved in CI.</what-built>

    Check each identity and record status (available / reserved / SQUATTED) + owner if taken:
    1. GitHub org: open https://github.com/axiam — is the org free or controllable?
    2. crates.io: `cargo search axiam-sdk` (or open https://crates.io/crates/axiam-sdk)
    3. npm: `npm view axiam-sdk` (404 == available)
    4. PyPI: open https://pypi.org/project/axiam-sdk/ (or `pip index versions axiam-sdk`)
    5. Maven Central: search https://central.sonatype.com for `io.axiam:axiam-sdk` (requires Sonatype namespace ownership of `io.axiam`)
    6. NuGet: open https://www.nuget.org/packages/Axiam.Sdk (or `nuget search Axiam.Sdk`)
    7. Packagist: open https://packagist.org/packages/axiam/axiam-sdk
    For each name that is TAKEN/squatted, decide and record a fallback (e.g. `axiamio-sdk`, `io.axiam.client:axiam-sdk`) so Phases 16-22 do not commit to an unusable name. Reserve the names you control now if possible.
  </how-to-verify>
  <resume-signal>Type "approved" with the availability table (and any fallback name decisions), or "blocked: <name> taken, fallback = <x>".</resume-signal>
