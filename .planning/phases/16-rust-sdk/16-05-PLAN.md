## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| incoming HTTP request ŌåÆ consumer's Actix app | Inbound tokens are untrusted until locally verified against JWKS |
| extractor ŌåÆ request context | Only verified identity may be injected into the request extensions |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-16-18 | Spoofing | forged token in cookie/Bearer header | high | mitigate | Local EdDSA signature + exp verification against cached JWKS before injecting identity; reject non-EdDSA alg (inherited from 16-02 verifier) |
| T-16-19 | Elevation of Privilege | stale verification cached beyond TTL | medium | mitigate | Verify per request; do not cache the decision beyond token TTL (┬¦10); exp checked each call |
| T-16-20 | Information Disclosure | token echoed in error body/logs | high | mitigate | Standardized JSON error body carries no token; `Sensitive<T>` discipline; extractor never logs the raw token (┬¦7/┬¦10) |
| T-16-21 | Spoofing | tenant confusion via header vs token | medium | mitigate | Identity (incl. tenant_id) is taken from the verified JWT claims, not from a request-supplied header (┬¦10) |
</threat_model>
