<objective>
Establish the TypeScript SDK build/test foundation and the dependency-free `core` module that
every transport layers on top of. This plan fills in the Phase-15 scaffold: replaces plain `tsc`
with tsup dual ESM+CJS multi-entry, wires vitest, generates gitignored proto stubs at build, and
implements the four `core` primitives shared across personas — error taxonomy (D-16), central
status->error mapper (D-17), `Sensitive<T>` redaction (D-26), CSRF token helpers, and the
single-flight refresh guard (D-07/D-13). Entry stubs for `.`/`/rest`/`/grpc`/`/amqp` are created so
later parallel plans never edit the same entry file.

Purpose: Lock the persona boundary and the shared behavioral core once, so REST/gRPC/AMQP/middleware
plans build against a stable, tree-shakeable foundation. Fixes the D-14 naming typo at the source.
Output: A buildable, testable `sdks/typescript/` package with a `core` module and empty transport
entry points, plus core unit tests (error mapping, Sensitive redaction, single-flight).
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| SDK code -> log/console/serializer | Token values could leak into diagnostic output; `Sensitive<T>` is the redaction boundary. |
| consumer bundler -> `/rest` entry | Node-only transports must not cross into a browser bundle (SC#1); enforced downstream but seeded here by a dependency-free core. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-17-01 | Information Disclosure | Sensitive<T> (core/sensitive.ts) | high | mitigate | Redact `[SENSITIVE]` across toString/toJSON/util.inspect.custom; raw value only via `expose()`; unit test asserts no raw leak on any of the three surfaces (D-26). |
| T-17-02 | Information Disclosure | error classes (core/errors.ts) | high | mitigate | Error messages/fields never embed raw token strings (§2 / D-16); AuthzError carries only action/resourceId, NetworkError carries cause (non-token). |
| T-17-03 | Tampering | error taxonomy drift REST vs gRPC | medium | mitigate | One central `errorMapper` (D-17) is the sole source both transports import; unit test pins every §2 row so drift cannot pass CI. |
| T-17-04 | Tampering | tree-shaking leak of Node transports | high | mitigate | Dependency-free core (no @grpc/grpc-js/amqplib import) + `splitting:false` + `sideEffects:false`; full end-to-end proof is the 17-06 bundle-grep gate (SC#1). |
