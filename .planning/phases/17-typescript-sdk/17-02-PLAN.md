<objective>
Implement the isomorphic REST + auth core at the root `.` / `/rest` entry (D-01/D-25): the
`AxiamClient` REST surface, cookie double-submit CSRF forwarding (D-05), reactive single-flight
401->refresh (D-07), the two-phase login/MFA discriminated union (D-18), authz via the FND-04 REST
endpoints (D-08), and idempotent-only retry with backoff/Retry-After (CF-01). This is a near-1:1
generalization of `frontend/src/lib/api.ts` — configurable baseUrl + tenant header, no app-store
coupling. It also defines the `SharedSession` object (cookie jar handle placeholder, CSRF token
store, single-flight guard, tenant) that the Node persona and middleware attach transports to (D-13).

Purpose: Deliver the browser-safe REST persona that satisfies SC#2 (browser authz over REST) and
SC#3 (single-flight + CSRF), and establish the shared session other transports compose with.
Output: A working REST client with login/verifyMfa/refresh/logout + can/checkAccess/batchCheck, an
axios instance with CSRF + refresh interceptors, retry policy, and msw-backed unit tests proving SC#2
and SC#3.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser JS -> AXIAM server (REST) | Untrusted state-changing requests cross here; CSRF double-submit is the mitigation. |
| SDK -> server auth endpoints | 401 responses can trigger refresh storms; single-flight guard bounds them. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-17-05 | Spoofing/Tampering | CSRF on state-changing REST calls | high | mitigate | Request interceptor forwards axiam_csrf cookie as X-CSRF-Token on POST/PUT/PATCH/DELETE (D-05); test asserts presence on POST, absence on GET. |
| T-17-06 | Denial of Service (self-inflicted) | thundering-herd refresh | high | mitigate | Reactive single-flight guard (D-07/§9); SC#3 test asserts exactly 1 refresh for 5 concurrent 401s; refresh-on-refresh barred by SKIP_REFRESH + §9.3. |
| T-17-07 | Information Disclosure | token in login/refresh result | high | mitigate | Public API exposes no access_token field (tokens via Set-Cookie only); grep + test assert absence (§7/D-16). |
| T-17-08 | Tampering | TLS bypass via customCa misuse | high | mitigate | customCa only ADDS a PEM CA (never disables verification); non-PEM input rejected at construction (§6); no insecure/rejectUnauthorized:false surface (17-06 lint gate enforces repo-wide). |
</threat_model>
