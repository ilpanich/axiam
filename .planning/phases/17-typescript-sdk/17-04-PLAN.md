<objective>
Implement the Node-only AMQP consumer (D-12 / §8): a byte-identical HMAC-SHA256 sign/verify pair
matching `crates/axiam-amqp/src/messages.rs`, the server-identical message DTOs, and a
closure-handler consumer that verifies-before-handler, owns the ack/nack loop, and on any
verification failure nacks-without-requeue and emits a security event. This is the TS port of the
already-tested Rust `verify_and_dispatch`/`consume`. The signing key is a required caller-supplied
`Sensitive<Buffer>` (no server endpoint provides it — confirmed against `axiam-amqp` config, which
sources it from `AXIAM__AMQP__SIGNING_KEY`).

Purpose: Deliver the AMQP half of the Node persona's transport coverage with the security-critical
verify-before-handler guarantee that a forged/tampered message never reaches consumer code.
Output: `axiam-sdk/amqp` entry with signPayload/verifyPayload, message types, and a `consume()`
function, plus unit tests proving valid/mismatched/missing-signature handling and nack-no-requeue.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| RabbitMQ message -> consumer handler | Messages may be forged/tampered/replayed; HMAC verify-before-handler is the boundary. |
| signing key -> logs | The per-tenant AMQP secret must never appear in logs; Sensitive<Buffer> + §8.4 event rules. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-17-09 | Tampering | AMQP message forgery/tampering | high | mitigate | HMAC-SHA256 verify-before-handler with constant-time compare; mismatch -> nack-no-requeue + security event; handler never sees unverified message (D-12/§8). |
| T-17-10 | Spoofing | missing-signature message | high | mitigate | Strict mode default nacks unsigned messages without requeue (§8.3); lenient mode is opt-in only. |
| T-17-11 | Information Disclosure | signing key / signature in logs | high | mitigate | Key is Sensitive<Buffer>; security event logs only timestamp/exchange/routingKey/tenant, never the HMAC or key (§8.4); test asserts absence. |
| T-17-12 | Tampering | nack argument transposition | medium | mitigate | Explicit `(msg, /* allUpTo */ false, /* requeue */ false)` + a test asserting requeue===false on every failure path (Pitfall 4). |
| T-17-13 | Tampering | pre-verify schema reordering | medium | mitigate | Canonical JSON via plain JSON.parse+delete+JSON.stringify; no schema validator before HMAC verify (Pitfall 5); key-order test in Task 1. |
