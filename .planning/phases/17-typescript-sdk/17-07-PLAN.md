<objective>
Close two of the four confirmed Critical security gaps from 17-VERIFICATION.md:

- **CR-02:** the single-flight refresh guard is a process-wide module singleton
  (`core/singleFlightRefresh.ts:9`), so concurrent 401s on DIFFERENT sessions cross-wire.
  Convert it to a per-session guard via a `createRefreshGuard()` factory and thread one guard
  per `SharedSession`/`NodeSession` through the REST interceptor and gRPC call-wrapper.
- **CR-03:** `middleware/verifyCore.ts` `authenticateRequest` only checks `claims.tenant_id`
  presence, never equality to a configured tenant, allowing a Tenant-A token to authenticate
  against a Tenant-B resource server in the same org (JWKS is org-wide). Add
  `tenantHeaderValue` to `VerifiableSession` and enforce claim equality.

These two gaps share NO files with each other, so they are two independent tasks in this plan.

Purpose: restore the SDK's per-session refresh correctness (D-13) and the project's core
per-tenant data-isolation guarantee for the resource-server middleware.
Output: a per-session refresh guard, tenant-enforcing middleware verify core, one regression
test per gap, and all 77 pre-existing tests still passing.
</objective>
