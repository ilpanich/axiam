<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| SDK → AXIAM REST API (TLS) | All auth/authz traffic crosses the network; certificate verification and tenant scoping happen here |
| Concurrent goroutines → shared token state | N goroutines observing an expired token may each attempt a refresh (thundering herd) |
| Consumer-supplied `*http.Client` → SDK transport | An override could drop the cookie jar or TLS verification if the SDK trusted it blindly |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-18-04 | Tampering / Information Disclosure | TLS transport in `client.go` | high | mitigate | `tls.Config{MinVersion: TLS13}`; only escape hatch is `WithCustomCA` (PEM → RootCAs); no `InsecureSkipVerify`; grep gate in `<verify>` (§6/SC#3) |
| T-18-05 | Elevation of Privilege | Consumer `WithHTTPClient` override dropping jar/TLS | high | mitigate | D-09: SDK re-applies its own `cookiejar` + TLS config over any supplied client; `TestClientOwnsCookieJarAndTLS_OverrideSafe` proves it |
| T-18-06 | Denial of Service (self-inflicted) | Concurrent 401s each firing a rotating single-use refresh | high | mitigate | §9 `sync.Mutex` single-flight guard with double-check; `TestRefreshGuard_SingleFlight -race` asserts exactly 1 refresh |
| T-18-07 | Spoofing | Missing tenant scope on requests | medium | mitigate | §5 tenant required at construction; `X-Tenant-ID` injected on every request; `TestTenantHeader_InjectedOnEveryRequest` |
| T-18-08 | Tampering (CSRF) | State-changing REST requests | medium | mitigate | §3 non-browser: capture `X-CSRF-Token` response header, echo on POST/PUT/PATCH/DELETE; `TestCSRF_CaptureAndForward` |
| T-18-09 | Information Disclosure | Raw tokens in error messages/logs | high | mitigate | Tokens wrapped in `Sensitive` (18-01); errors constructed via 18-01 mappers that never embed token strings; logger OFF by default (CF-02) |
</threat_model>
