<objective>
Implement local JWKS verification (D-06/§10) and the gRPC transport (§5/§6, SC#3 gRPC half). The JWKS verifier uses `jwx/v3`'s `jwk.Cache` bound to the confirmed org-wide `{baseURL}/oauth2/jwks` path, enforces an explicit EdDSA allowlist before verification, and force-refetches on an unknown `kid`. The gRPC client uses `grpc.NewClient` + `credentials.NewTLS` (strict TLS, no insecure surface) with a sync-safe auth/tenant interceptor, calling `CheckAccess`/`BatchCheckAccess` over the committed stubs.

Purpose: The JWKS verifier is the shared local-verify primitive the middleware (Plan 05) consumes; the gRPC transport completes the SC#3 no-TLS-bypass invariant across both transports.
Output: `internal/jwks/` (verifier + claims), `grpc/` (tls, interceptor, client), and tests.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| SDK → AXIAM gRPC AuthorizationService (TLS) | Low-latency authz checks cross the network; TLS verification + tenant scoping happen here |
| JWKS endpoint → local verifier | The org-wide JWKS is fetched and cached; the verifier must reject wrong-alg tokens and not trust the token's own header |
| Token cache → gRPC interceptor | The interceptor reads the access token synchronously on every RPC; a blocking lock would stall the RPC path |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-18-14 | Tampering / Information Disclosure | gRPC TLS transport | high | mitigate | `credentials.NewTLS(tls.Config{MinVersion:TLS13})`; only escape hatch is `WithCustomCA`; grep gate extended to `WithInsecure`/`insecure.NewCredentials`/`grpc.Dial` (Pitfall 1) |
| T-18-15 | Spoofing / Tampering | Algorithm-confusion (alg:none/HS256 against an EdDSA key) | high | mitigate | Explicit EdDSA allowlist checked BEFORE `jws.Verify`; token's own alg header never selects the algorithm (`TestJWKS_RejectsWrongAlg`) |
| T-18-16 | Elevation of Privilege | Stale JWKS after key rotation (unknown kid) | medium | mitigate | Forced single refetch + retry on unknown kid via `jwk.Cache.Refresh` (`TestJWKS_UnknownKidRefetchesOnce`) |
| T-18-17 | Denial of Service | Blocking `.Lock()` in the gRPC interceptor stalling every RPC | medium | mitigate | Interceptor reads the token via a non-blocking accessor closure; no refresh-mutex lock in the closure (Anti-Patterns) |
| T-18-18 | Spoofing | Missing tenant metadata on gRPC calls | medium | mitigate | §5 `x-tenant-id` metadata injected on every RPC (`TestInterceptor_InjectsBearerAndTenant`) |
</threat_model>
