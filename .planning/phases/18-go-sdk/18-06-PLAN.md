<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Pull request → CI gates | PRs must be blocked if they introduce a TLS-bypass surface or stale committed stubs |
| Release tag → module publish | Publishing is a side-effecting, credential-adjacent step that must run only on maintainer tags, never on PRs |
| Committed stubs → proto source of truth | Drift between committed `internal/gen` stubs and the protos is a tampering/stale-contract risk |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-18-23 | Tampering / Information Disclosure | A PR introducing a TLS-bypass surface | high | mitigate | CI `tls-bypass-gate` runs the extended grep (`InsecureSkipVerify|WithInsecure\(|insecure\.NewCredentials\(`) and fails on any match (SC#3, Pitfall 1) |
| T-18-24 | Tampering | Committed gRPC stubs drifting from the protos | medium | mitigate | `buf-drift-check` regenerates and `git diff --exit-code sdks/go/internal/gen` fails on drift (D-01) |
| T-18-25 | Information Disclosure | Publish credentials exposed to PR-triggered jobs | high | mitigate | `publish` job gated to `refs/tags/sdks/go/v*` pushes; never runs on `pull_request`, so no secret is reachable from PR jobs (mirrors Rust T-16-26) |
| T-18-26 | Tampering | Unpinned GitHub Actions (supply-chain) | medium | mitigate | All actions SHA-pinned per the repo-wide convention (Phase 12 SEC-057) mirrored from the Rust workflow |
| T-18-SC | Tampering | npm/pip/cargo/go installs | high | mitigate | The three GO-01 deps are `[VERIFIED: Go module proxy]` (RESEARCH Package Legitimacy Audit); no `[ASSUMED]`/`[SUS]` packages requiring a blocking checkpoint |
