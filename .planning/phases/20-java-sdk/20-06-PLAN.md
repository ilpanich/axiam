## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| incoming HTTP request (consumer app) -> SecurityContext | bearer/cookie token is untrusted until locally verified |
| org-wide JWKS -> tenant-scoped authorization | a signed token for another tenant must be rejected |

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |

| T-20-06 | Spoofing/Tampering | algorithm confusion on the resource-server path | high | mitigate | local verify uses the EdDSA-pinned `JwksVerifier` (20-04) |
| T-20-10 | Spoofing | expired-but-signed token accepted | medium | mitigate | explicit `exp` check in the filter (resource-server trust boundary rejects expired) |
