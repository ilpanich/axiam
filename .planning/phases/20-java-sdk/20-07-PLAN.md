<objective>
Deliver the AMQP event consumer (D-13, §8): a callback `consume(...)` that verifies each delivery's HMAC-SHA256 (via 20-02's proven `Hmac.verify`) BEFORE invoking the caller's handler, owns the manual-ack/nack loop, and enforces the exact §8 decision matrix — handler success -> ack; retryable handler failure -> nack WITH requeue; HMAC-fail / parse-fail / `ErrDrop` sentinel -> nack WITHOUT requeue + a security log entry (that never contains the HMAC value). It relies on the RabbitMQ Java client's built-in automatic recovery (connection + topology) rather than a hand-rolled reconnect loop.

Purpose: The AMQP subsystem must never hand an unverified message to consumer code; the ack/nack matrix and no-HMAC-in-logs rules are the §8 tamper/replay and info-disclosure controls.
Output: `AmqpConsumer` + its ack/nack-matrix test.
</objective>
