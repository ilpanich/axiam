<objective>
Deliver the gRPC transport (D-11/D-12): the `GrpcAuthzClient` wrapping one long-lived `grpc-netty-shaded` `ManagedChannel` with both blocking and `CompletableFuture`-adapted async surfaces for `CheckAccess`/`BatchCheckAccess`, a per-call default deadline (overridable), a non-blocking metadata-injecting `AuthClientInterceptor` (Bearer + `x-tenant-id`), UNAUTHENTICATED -> single-flight-refresh (the SAME guard REST uses, via the 20-05 seam) -> retry-once, and central `ErrorMapper.fromGrpcStatus` error mapping. The channel is built from `AxiamClient`'s strict-TLS/customCa config (no bypass) and shut down in the client's `close()`.

Purpose: Authz is a latency-sensitive hot path; the gRPC transport must share the ONE refresh guard (D-07) and never weaken TLS.
Output: gRPC client + interceptor + tests built on the codegen stubs (20-01) and the REST session/guard seam (20-05).
</objective>
