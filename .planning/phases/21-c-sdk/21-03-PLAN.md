## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Untrusted JWT â†’ SDK verifier | Token header/payload/signature are attacker-controlled until verified against the org JWKS. |
| Org-wide JWKS â†’ per-tenant authorization | A valid signature proves org membership, NOT the configured tenant â€” an extra claim check is required. |
| Concurrent callers â†’ refresh endpoint | A token-refresh stampede can exhaust the single-use refresh token and trigger server rate limits. |
