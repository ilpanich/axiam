<objective>
Implement local EdDSA/Ed25519 JWKS verification (D-08) and the `LoginResult` readonly DTO (D-09).
`JwksVerifier` sources keys via OIDC discovery, TTL-caches them, rotates on an unknown `kid`, pins
`alg=EdDSA` before key lookup (Pitfall 5 / alg-confusion), and enforces the post-signature
`tenant_id` claim check (Pitfall 3 / cross-tenant replay) — the two security invariants every
sibling SDK independently confirmed.

Purpose: give the framework middleware (wave 5) a fast local-verify path and the client a proactive
refresh basis, without a server round-trip per request and without the CachedKeySet PSR chain.
Output: `src/Auth/JwksVerifier.php`, `src/Auth/LoginResult.php`, `tests/JwtVerifyTest.php`, and
committed Ed25519 crypto fixtures.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| attacker-crafted JWT → SDK verifier | Untrusted `alg`/`kid`/claims cross into local verification |
| org-wide JWKS → tenant-scoped SDK | A validly-signed token for another tenant crosses the tenant boundary |

| T-22-04 | Spoofing | `JwksVerifier::verify` alg handling | high | mitigate | Pin `alg=EdDSA` from the header BEFORE key lookup; reject `none`/other algs (Pitfall 5) |
