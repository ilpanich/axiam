# Phase 22 Plan 02: JWKS EdDSA Verifier + LoginResult DTO + Crypto Fixtures Summary

**Local EdDSA/Ed25519 JWKS verifier (`JwksVerifier`) enforcing alg-pin-before-key-lookup and post-signature tenant_id scoping via OIDC discovery + a hand-rolled TTL cache, plus a readonly `LoginResult` DTO with a `Sensitive`-typed challenge token, both proven against real committed Ed25519/JWKS/JWT crypto fixtures.**

## Accomplishments
- Generated a real Ed25519 keypair via `ext-sodium`'s `sodium_crypto_sign_keypair()` and committed three fixtures (`tests/Fixtures/ed25519_{jwks,signed_jwt,keypair}.{json,txt}`), self-verified before commit via `JWK::parseKeySet()` + `JWT::decode()` — the committed JWT round-trips against the committed JWKS through `firebase/php-jwt`, and the JWKS document matches `sdks/openapi.json`'s `Jwk` schema exactly (`kty=OKP`, `crv=Ed25519`, `alg=EdDSA`, `kid`, `x`, `use=sig`)
- `sdks/php/tests/Fixtures/verify_fixture.php`: a standalone, non-PHPUnit CLI check script (`php tests/Fixtures/verify_fixture.php`) that parses the committed JWKS, decodes the committed JWT, and prints the `tenant_id` claim — the exact command the plan's Task 1 `<verify>` block specifies
- `Axiam\Sdk\Auth\JwksVerifier::verify(string $jwt, string $expectedTenantId): ?array` — pins the header `alg` to `EdDSA` **before** any key lookup (Pitfall 5 / T-alg-confusion), resolves `jwks_uri` via `GET /.well-known/openid-configuration` (falling back to `{baseUrl}/oauth2/jwks`), TTL-caches `JWK::parseKeySet()` output, refetches **exactly once** on an unknown `kid` before failing closed (D-08 / T-DoS), and rejects any token whose `tenant_id` claim doesn't match `$expectedTenantId` **after** signature verification succeeds (Pitfall 3 / T-cross-tenant) — no `CachedKeySet` (Pitfall 2), guards `extension_loaded('sodium')` with a clear `AxiamException`, never throws on malformed/short/non-3-part attacker-controlled token input
- `Axiam\Sdk\Auth\LoginResult` — `final readonly class` (D-09) exposing `mfaRequired: bool` plus `?Sensitive $challengeToken`, `?string $userId`, `?string $tenantId` — no raw token ever exposed as a plain public property (CONTRACT §7, mirrors the Java SDK's 20-03 `challengeToken` decision)
- `sdks/php/tests/JwtVerifyTest.php` — 7 tests / 20 assertions: alg-pin rejection for both `alg=none` and `alg=RS256` (each proven via an **empty** `MockHandler` queue — a key lookup would throw "Mock queue is empty" instead of returning `null`), tenant-mismatch rejection on both the original fixture (different `expectedTenantId`) and a freshly re-signed wrong-tenant token, happy-path claims return, unknown-kid single-refetch (proven via `Middleware::history` request-count assertion, `assertCount(2, ...)` = exactly 1 discovery + 1 jwks fetch), and malformed/empty/short token input never throwing

## Decisions Made
- firebase/php-jwt's EdDSA sign/verify internals expect the key argument as a **standard** (non-urlsafe) base64-encoded string (they `explode("\n", $key)` and `base64_decode()` the last line) — the committed `ed25519_keypair.json` stores raw key bytes as base64url for portability/inspection, and `JwtVerifyTest`'s re-signing helper explicitly re-encodes to standard base64 before calling `JWT::encode()`. The JWKS `x` field itself stays base64url per RFC 7517 (unaffected — `JWK::parseKey()` internally calls `JWT::convertBase64urlToBase64()`).
- `ensureFresh()` swallows discovery/JWKS-fetch `Throwable`s and leaves the existing (possibly still-`null`) cache untouched on failure, rather than propagating an exception up through `verify()` — the existing "unknown kid after refetch → `null`" fail-closed path already covers a fetch failure without needing a second exception-handling branch in `verify()` itself.
- Committed `verify_fixture.php` (not explicitly listed in the plan's `files_modified` frontmatter, but required by the plan's own Task 1 `<verify><automated>` command) as a standalone CLI script rather than folding the check into a PHPUnit test, so the exact command the plan specifies (`php tests/Fixtures/verify_fixture.php`) is directly runnable and exits 0.
