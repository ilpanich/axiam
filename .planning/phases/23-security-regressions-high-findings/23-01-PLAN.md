<objective>
Close SEC-003 and SEC-026b: attach authentication to the gRPC `UserService` and `TokenService`, cross-validate tenant identity from verified JWT claims (not the request body), and route the gRPC credential-check through the same failed-login/lockout accrual the REST login path uses (D-06).

Purpose: Today `user_svc` and `token_svc` are registered with plain `::new(...)` — zero interceptor, zero auth (server.rs:69-70). Any unauthenticated mesh peer can call `GetUser`, `ValidateCredentials`, `IntrospectToken`, and `ValidateCredentials` never accrues lockout state on a wrong password — an unmetered credential-check oracle. `AuthorizationService` already solves the auth+cross-validate problem (authorization.rs:69-116); this plan replicates that chokepoint across the remaining two services and unifies lockout accrual per D-06.

Output: A shared `axiam_auth::lockout` helper; `UserServiceServer`/`TokenServiceServer` wrapped with `AuthInterceptor`; claims cross-validation on `GetUser`/`ValidateCredentials`; and reject-without-token + cross-tenant + lockout-accrual negative tests.
</objective>
