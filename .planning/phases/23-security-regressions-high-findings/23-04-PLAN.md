<objective>
Close SEC-005 (XML Signature Wrapping): bind the verified XML signature to the assertion actually consumed by `handle_saml_response`, and enforce Destination + InResponseTo on the authenticated ACS path.

Today `verify_signature` (saml.rs:378, implemented at 579-591 via `samael::crypto::verify_signed_xml(xml_bytes, &der, Some("ID"))`) returns only `Result<(), _>` — it proves *some* valid `<ds:Signature>` exists somewhere in the document but never tells the caller WHICH element ID it verified. `handle_saml_response` then independently reads `response.assertion` (saml.rs:436) — a scalar `Option<Assertion>`. An attacker can keep the original signed assertion intact somewhere in the tree (so the lone-signature check still passes) and inject a second, forged, unsigned `<Assertion>` sibling that the deserializer binds to `response.assertion`. Nothing today binds "the element that was cryptographically verified" to "the element whose claims get trusted."

Purpose: The research-verified fix (23-RESEARCH Pattern 5) is a hand-written raw-XML introspection check using `libxml` (already resolved transitively via samael 0.0.19's `xmlsec` feature) — NOT a samael version bump and NOT hand-rolled reference-string matching (23-RESEARCH Anti-Patterns forbids regex/string XML parsing). samael STAYS at 0.0.19; `verify_signature` is UNCHANGED; a new binding step is inserted after the assertion is read and before its claims are trusted. Because the exact `libxml` XPath/`Context` API surface is flagged LOW-confidence in research (docs.rs returned 403; Assumption A2), a dedicated verification SPIKE (Task 1) confirms the call shape against the vendored crate source BEFORE any implementation.

Scope boundary (per 23-CONTEXT <deferred> and 23-RESEARCH Deferred Ideas): the XSW signature↔assertion binding, the authenticated-ACS Destination check, and the authenticated-ACS InResponseTo presence requirement are IN SCOPE. Full SAML `Recipient`/`SubjectConfirmationData` validation beyond this XSW-binding + Destination + InResponseTo minimum is OUT OF SCOPE — tracked as the SEC-005 residual, recorded explicitly in Task 3, NOT silently dropped.

Output: libxml pinned as a verified direct dep of axiam-federation; the libxml call shape confirmed against vendored source; XSW-safe signature↔assertion binding; authenticated-path Destination + InResponseTo; XSW/Destination/InResponseTo negative tests.
</objective>
