<objective>
Close SEC-015 / CQ-F05: make logout actually revoke the caller's session. The revocation machinery is already correct — `AuthService::logout` hard-deletes the session row (service.rs:580-582 → session.rs:186-198), and `SessionValidator` re-checks session liveness on every request (extractors/auth.rs:25-60, wired at main.rs:342). The ONLY defect is that the `logout` handler requires a `LogoutRequest { session_id }` body (auth.rs:104-106,351) that the frontend doesn't send correctly (`Topbar.tsx:89-98` posts `{}`), so the endpoint 400s before revocation ever runs — logout is currently a client-side-only no-op with a swallowed error.

Purpose: Per D-03, revoke server-side from the authenticated JWT `jti` (`AuthenticatedUser.session_id`, already == jti), drop the request body entirely (no client-supplied session_id, no IDOR surface), and keep clearing all three cookies. This is deliberately a SMALL fix — do NOT build token blocklists, JWT versioning, or new session-tracking (Pitfall 3); the per-request liveness check already handles replay.

Output: a body-less logout handler, removal of the redundant DTO, a backend replay-after-logout negative test, and a frontend fix + Playwright logout spec.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client logout request → session revocation | The session to revoke must come from the verified JWT, never a client-supplied id |
| Old auth cookie → post-logout request | A replayed cookie after logout must not re-authenticate |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-23-05-A | Repudiation / Spoofing | logout no-op (handler 400s before revocation) | high | mitigate | Body-less handler revokes from jti; SessionValidator rejects replay; replay-after-logout 401 test (Task 1) |
| T-23-05-B | Elevation of Privilege (IDOR) | client-supplied session_id (LogoutRequest) | medium | mitigate | Remove the body entirely; session derived from verified jti only (D-03) |
| T-23-05-C | Spoofing | surviving cookies after logout | high | mitigate | All three cookies (access/refresh/csrf) cleared server-side; e2e unauthenticated-after-logout assertion |

ASVS L1: V3 Session Management (server-side invalidation on logout, all-cookie clearing, no client-supplied session identifier), V2 Authentication (per-request session-liveness re-check). Block-on severity: high — replay and surviving-cookie threats mitigated with proving tests.
</threat_model>
