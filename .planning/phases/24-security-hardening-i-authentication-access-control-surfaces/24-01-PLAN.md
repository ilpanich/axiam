<objective>
Close the TOTP replay window (SECHRD-01 / SEC-008) by making the last-used-step advance an atomic database compare-and-set, recording the step that ACTUALLY matched (including the -1 skew step), and seeding `totp_last_used_step` at enrollment-confirm time.

Purpose: Today `update_totp_step` runs an unconditional `UPDATE ... WHERE tenant_id = $tenant_id` with no step guard, so two concurrent submissions of one valid code both succeed, and a skew-accepted code can be replayed in a later wall-clock step because `current_step` (not the matched step) is recorded. This is a Spoofing / Elevation-of-Privilege second-factor bypass.
Output: An atomic CAS in `update_totp_step`, matched-step recording in `verify_code_with_replay_check`, enrollment-confirm seeding, and a concurrency negative test proving exactly-one-success.
</objective>

Bind `id`, `tenant_id`, `step`; call `.check()` then `.take(0)` into a row vec; an empty vec means the guard did not match (replay or a concurrent winner).
