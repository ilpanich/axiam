<objective>
Close the secret-material-in-memory side-channel (SECHRD-12 / T19.24): wrap the peppered-password buffer with `zeroize` and the pepper value with `secrecy`, wiped before return. Also relocate the shared `DUMMY_HASH` constant into the password module so plan 24-09's constant-time reset can reuse the identical constant without drift.

Purpose: A peppered password concatenation currently lingers in process memory (recoverable via core/memory dump — Information Disclosure), and the pepper is a plain `Option<String>` that can be accidentally `Debug`-printed or cloned into logs. This plan introduces `zeroize`/`secrecy` and fixes every `config.auth.pepper` call site so the workspace still compiles.
Output: `Zeroizing<String>` buffers in `hash_password`/`verify_password`, `AuthConfig.pepper: SecretString`, `DUMMY_HASH` moved to `pub(crate)` in the password module, and `expose_secret()` at each `&str` boundary — Argon2 params untouched.
</objective>
