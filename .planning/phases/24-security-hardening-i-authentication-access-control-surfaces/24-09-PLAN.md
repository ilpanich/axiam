<objective>
Close the remaining auth-recovery side-channels (SECHRD-12, decision D-04 + SEC-028 residual, resolves RESEARCH Open Question 2): make the password-reset initiation constant-time on the ineligible/unknown/federated branches (dummy Argon2 hash + async wait, mirroring the live SEC-026 login pattern), block reuse of the CURRENT password on the unauthenticated reset path, and seed initial passwords into history so the reuse check has data.

Purpose: `initiate_reset` returns `Ok(None)` on unknown/federated accounts WITHOUT doing the Argon2 work the valid branch does — a timing side-channel enabling user enumeration (Information Disclosure). And `confirm_reset` evaluates history BEFORE the pre-reset hash is in history, so a user can reset to their CURRENT password (RESEARCH Pitfall 4). Federated/admin-created and bootstrap users may also have no seeded history at all (Pitfall 5).
Output: `crypto_semaphore` plumbed into `PasswordResetService`, a `dummy_hash_wait` on both `Ok(None)` branches, an explicit current-password comparison in `confirm_reset`, a `create_with_consent` history seed, and timing + current-password-reuse negative tests.

DEPENDS ON: plan 24-05 (relocated `pub(crate) DUMMY_HASH` in the password module + `AuthConfig.pepper: SecretString`); plan 24-08 (bootstrap history-seed write path — this plan seeds the create_with_consent write path); plan 24-01 (user.rs ordering).
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| unauthenticated caller → password-reset initiate/confirm | Attacker probes response timing to enumerate valid accounts, or tries to reset to the current password |
| reset service → Argon2 (crypto_semaphore) | All password hashing must be CPU-isolated and constant-time-equivalent |

## STRIDE Threat Register (ASVS V2 Authentication, V6 Cryptography)

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-24-91 | Information Disclosure (user enumeration) | initiate_reset Ok(None) branches | high | mitigate | Dummy Argon2 + async wait mirroring SEC-026 on both ineligible branches; statistical timing test (Task 1) |
| T-24-92 | Spoofing / weak-credential reuse | confirm_reset current-password gap (Pitfall 4) | medium | mitigate | Explicit `verify_password(new, current_hash)` rejection, semaphore-gated (Task 2) |
| T-24-93 | Business-logic gap | unseeded initial-password history (Pitfall 5) | medium | mitigate | Seed create_with_consent (Task 3) + bootstrap path (plan 24-08); federated paths excluded |
</threat_model>
