<objective>
Create ONE shared, fresh-per-request, IP-pinning SSRF-guarded HTTP fetch helper in `axiam-federation` and route the three federation outbound-fetch call sites (JWKS, OIDC discovery + token exchange, SAML metadata) through it. This generalizes the existing `is_private_jwks_ip` guard (`jwks_cache.rs:225`) into one reusable function and — critically — adds IP **pinning**, which no existing federation guard does today (it validates the resolved IP but lets reqwest re-resolve DNS at send time, leaving an open DNS-rebind TOCTOU window).

Purpose: Close the SSRF / DNS-rebind trust boundary for all IdP-supplied URLs so the fetch fails closed against internal-address targets and redirect bypasses (SECHRD-02, per D-01a/b/c).
Output: `crates/axiam-federation/src/ssrf.rs` + rewired federation fetch call sites + two federation-crate negative tests (loopback token_endpoint rejected; redirect-to-internal rejected).
</objective>

    Add two `#[cfg(test)]` unit tests: `ssrf_rejects_loopback_token_endpoint` (assert `resolve_and_pick("localhost"/127.0.0.1 host, _, allow_private=false)` returns `Err(Blocked)`); `ssrf_rejects_redirect_to_internal` (spin a loopback wiremock/`tokio` server via the `allow_private=true` seam that returns 302 with `Location` pointing at `10.0.0.5`, assert `guarded_fetch` returns `Err(Blocked)` on the redirect hop — proving the hop is re-validated, not followed).
    No `unwrap()`/`expect()` on any resolve/build/send path — every failure returns an `SsrfError` variant (fail closed).
