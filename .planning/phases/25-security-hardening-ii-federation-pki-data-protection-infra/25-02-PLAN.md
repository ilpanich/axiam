<objective>
Fold `axiam-api-rest` webhook delivery onto the shared `axiam_federation::ssrf` guard so the validated `IpAddr` is pinned into the connection. Today `webhook.rs::resolve_and_validate_host` resolves and validates the IP but then `client.post(&webhook.url)` re-resolves DNS independently through reqwest's normal path — a genuine, currently-open DNS-rebind TOCTOU (RESEARCH Pitfall 1: the CONTEXT claim "already re-resolves+pins from Phase 11" is only half true — it re-resolves per retry, it does not pin per send).

Purpose: Close the webhook half of the SSRF/DNS-rebind boundary using the exact same shared guard as federation (no third copy of the IP-classification logic) (SECHRD-02, D-01a/c).
Output: rewired webhook delivery loop + `webhook_pins_resolved_ip` integration test proving no second resolution happens at send time.
</objective>
