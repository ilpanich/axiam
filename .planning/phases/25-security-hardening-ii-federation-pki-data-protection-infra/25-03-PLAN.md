<objective>
Add the immediate issuing-CA status/validity gate to `axiam-pki::mtls.rs`. Today the leaf device cert's `status`/`not_before`/`not_after` are checked (`mtls.rs:58-66`) and the issuing CA is fetched (`get_by_issuer_id`, `:74-83`) — but the CA's own status/validity is never asserted before `verify_signature`. A revoked or expired issuing CA is therefore still trusted for device-cert auth. This plan mirrors the existing leaf-cert check onto the issuer (D-02: immediate issuing CA only; full chain-walk is deferred for AXIAM's flat org/tenant-CA→device hierarchy).

Purpose: Make mTLS device auth fail closed against a revoked/expired issuing CA (SECHRD-05).
Output: the issuer status/validity check + two negative tests (revoked CA, expired CA).
</objective>
