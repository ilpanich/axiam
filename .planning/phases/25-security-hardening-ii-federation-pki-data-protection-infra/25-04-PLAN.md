<objective>
Build the DB-layer prerequisites for GDPR erasure durability and export correctness (SECHRD-06): (1) a new `SessionRepository::list_by_user` read method so the export can carry real (metadata-only) session rows instead of a hardcoded `[]`; (2) widen `export_job::has_pending_for_user`'s dedup filter to also block `ready`-undownloaded and `failed` jobs; (3) a DB `UNIQUE` index on `erasure_proof.user_id` so a retried erasure cannot write a duplicate proof.
