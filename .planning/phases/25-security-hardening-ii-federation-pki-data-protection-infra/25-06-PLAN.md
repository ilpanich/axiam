<objective>
Fix the authenticated account-linking OIDC callback (`handlers/federation.rs:595-648`) to derive the expected nonce from server-side login state, not the request body (SECHRD-07 / D-04). Today `oidc_authorize`/`oidc_callback` take `req.state`/`req.nonce` from the JSON body, validate only non-emptiness, and pass `req.nonce` straight into `handle_callback` — so a client-supplied nonce is trusted. An exact working reference already exists 550 lines away in the same file: `oidc_start_public`/`oidc_callback_public` (`:1106-1300`) persist a server nonce in `FederationLoginState` and read it back on callback. This plan replicates that plumbing on the authenticated path — the single lowest-risk fix in the phase.

Purpose: Close the OIDC nonce-replay hole on the account-linking path (SECHRD-07).
Output: nonce-from-state plumbing on `oidc_authorize`/`oidc_callback` + a replay negative test.
</objective>
