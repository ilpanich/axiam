<objective>
Promote AMQP message signing from optional/fail-open to mandatory and per-tenant (SECHRD-08 / D-05a/b/c). Today `audit_consumer.rs` and `authz_consumer.rs` both have a `warn!`-and-process branch when `signing_key` is `None` (and even when a signature is invalid), and `AmqpConfig::signing_key` documents a "migration mode" `Option`. This plan: adds the `hkdf` crate; adds `derive_tenant_key` (HKDF-SHA256, domain-separated + versioned) in `messages.rs`; wires both consumers to derive the per-tenant subkey from the message's `tenant_id` + `key_version` and verify (rejecting unsigned/invalid — no warn-and-process); and makes the config resolve to a mandatory `Vec<u8>` with a documented dev-key default (prod fails closed if unset). Scope is `AuthzRequest` + `AuditEventMessage` only — mail is NOT signed (RESEARCH Pitfall 5).

Purpose: A leaked/compromised tenant-A signing capability cannot forge tenant-B audit/authz events; no unsigned-message bypass exists (SECHRD-08).
Output: HKDF per-tenant keying + mandatory config + removed fail-open branches + cross-tenant + unsigned negative tests.
</objective>
