<objective>
Close the federation/PKI secret-leak class at the type level (SECHRD-09 / D-06). `FederationConfig` (`axiam-core/src/models/federation.rs`) derives `Debug, Serialize` with ZERO `skip_serializing` — it fully round-trips `client_secret`/`client_secret_ciphertext`/`client_secret_nonce`/`client_secret_key_version` through both `Serialize` and `Debug`. Additionally, PATTERNS.md flags that `CaCertificate` still leaks its `encrypted_private_key` via the derived `Debug` even though it has `skip_serializing` (which only affects `Serialize`, not `{:?}`). This plan applies the existing `certificate.rs:61-63` `skip_serializing` convention to `FederationConfig`'s four secret fields, replaces the derived `Debug` on BOTH `FederationConfig` and `CaCertificate`/`GeneratedCaCertificate` with a manual redacting impl, and narrows the `SELECT *` in `federation_config.rs::list()` so encrypted columns aren't hydrated for list views.

Purpose: Federation/PKI secrets are never serialized or printed anywhere (SECHRD-09).
Output: skip_serializing + redacting Debug on both models + narrowed list projection + a non-serialization/non-print negative test.
</objective>
