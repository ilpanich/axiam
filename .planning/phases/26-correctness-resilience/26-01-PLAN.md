<objective>
Fix CORR-01: the gRPC governor quota is inverted. `tower_governor`'s `GovernorConfigBuilder::per_second(n)` sets the replenish PERIOD to n seconds ("one token every n seconds"), not "n tokens/sec" — with the default `grpc_authz_per_sec=100` this throttles the mesh to ~1 token / 100 s, and raising the rate makes it slower. Reconstruct the quota via `governor::Quota::per_second` directly (D-01) and add a sustained-throughput regression test (D-02).

Purpose: The control plane's gRPC authz path must actually pass traffic at the configured rate; a barely-throttling governor is both a functional defect and a self-inflicted DoS enabler.
Output: A corrected `build_grpc_governor_layer` and a sustained-throughput test that would fail against the inverted construction.
</objective>
