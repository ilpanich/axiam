<objective>
Fix CORR-02: the 4-week SurrealDB root-token TTL with no renewal is an uptime ceiling — once the cached JWT expires, every request 401s until the process restarts. Add (D-03) BOTH a proactive periodic re-`signin` task (fires well inside the TTL, D-04) AND a reactive reconnect-on-auth-error safety net, derive the refresh interval from the TTL (`AXIAM__DB__TOKEN_REFRESH_FRACTION`, default ~0.6, D-20), and make `health_check` return Unhealthy on auth expiry (D-05). Keep the design forward-compatible with PERF-04's reconnect hook (Phase 27, same file).

Purpose: Keep the control plane's DB layer alive past the token TTL without a restart, and give the readiness probe a true signal when auth is unrecoverable.
Output: A renewing/reconnecting `DbManager` and a resilience test.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| DbManager -> SurrealDB (root session) | The service-account (root) bearer token authenticates every DB request; its renewal path must not leak or mis-handle the token |
| health_check -> readiness probe | An incorrect health signal can either mask an outage or cause false unavailability |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-26-02-01 | Information Disclosure | re-signin/reconnect logging | high | mitigate | No `tracing` statement logs the bearer token, JWT, or root password; only outcome (ok/err + non-secret context) is logged (Task 1/2 acceptance) |
| T-26-02-02 | Spoofing/Elevation | reactive reconnect on a genuinely revoked credential | medium | mitigate | health_check reports Unhealthy on ANY auth failure (not just expiry), so a revoked-credential state alarms rather than silently retrying forever (D-05, V3) |
| T-26-02-03 | DoS | ambiguous authenticated handle after failed re-signin | medium | mitigate | Proactive path re-signs only the still-valid handle; reactive path builds a fresh connection and swaps atomically — no half-authenticated handle is left serving requests |
| T-26-02-SC | Tampering | cargo installs | low | accept | No new packages this phase (RESEARCH Package Legitimacy Audit: none) |
</threat_model>
