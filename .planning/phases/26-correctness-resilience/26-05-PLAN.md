<objective>
Fix CORR-05 backend half (CORR-05a): the `/auth/me` DTOs (`MeResponse`/`LoginUserInfo`) do not emit `tenant_slug`/`org_slug`, so after a hard reload the frontend (which already reads these fields defensively) cannot restore the selected tenant. Add the two optional slug fields (D-14) and populate them in the `me` handler with graceful degradation (D-15), plus in the fresh-login response path. This is the backend precedent that CORR-05b (plan 26-08, frontend restore + MFA-setup) depends on.

Purpose: Give the frontend the org/tenant slugs it needs to restore tenant context on reload, without ever failing `/auth/me` when a slug can't be resolved.
Output: The two new DTO fields, populated in both response paths, with a backend test asserting serialization.
</objective>
