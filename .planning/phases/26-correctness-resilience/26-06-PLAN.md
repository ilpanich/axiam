<objective>
Fix CORR-06: three residual frontend correctness defects, each with a proven in-tree pattern. VerifyEmailPage's `cancelled`-closure survives StrictMode's mount→unmount→remount and double-fires the single-use verification token, surfacing a false "failed" (D-17). DashboardPage's user-count query key `["users",1,""]` collides with UsersPage's page-1/no-filter key, cross-contaminating the cache (D-18). The org-settings `SettingsTab` re-seeds form state on every refetch/refocus, silently discarding in-progress edits (D-19). Apply the codebase's existing `useRef` once-guard idiom, a distinct query key, and an init-guard + dirty-tracking + navigate-away guard respectively.

Purpose: Eliminate three user-visible correctness bugs in the auth-verify, dashboard-count, and org-settings-edit flows.
Output: The three corrected pages plus targeted vitest coverage.
</objective>
