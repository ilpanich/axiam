<objective>
Add a hand-rolled, process-wide circuit breaker around `check_hibp` (D-01/D-02) that
fails open (`Ok(None)`) and — critically — skips the HTTP request (and its 5s timeout)
entirely once tripped, so a credential-stuffing burst against a downed/rate-limiting
HIBP endpoint does not starve legitimate auth flows (D-03/D-04). Pre-size the
`check_complexity` violation `Vec` to capacity 5 (D-05).

Purpose: Close PERF-01. The breaker's value is avoiding wasted 5s timeouts under
sustained failure — `check_hibp` is ALREADY fail-open on every error branch, so this
does NOT add fail-open behavior; it adds burst-protection (RESEARCH Pitfall 4).
Output: `HibpBreaker` state machine + config knobs + pre-sized complexity Vec, all
unit-tested.
</objective>
