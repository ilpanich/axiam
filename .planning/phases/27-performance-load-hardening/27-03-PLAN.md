<objective>
Apply the uniform hand-rolled JWKS single-flight guard (D-08) to the Go, Java, and C#
SDK verifiers so a concurrent invalid-`kid` cache-miss burst collapses to exactly one
network fetch (D-09). Each guard wraps only the fetch/refresh — never the cryptographic
verification (`lestrrat-go/jwx/v3` / Nimbus / BouncyCastle).

Purpose: Close PERF-03 for 3 of the 7 SDKs. Note two of these delegate to libraries
with their own possible internal coalescing (jwx `jwk.Cache`, Nimbus `RemoteJWKSet`);
D-08 requires an explicit guard regardless so the guarantee does not depend on
unverified library internals. C# today has ZERO synchronization.
Output: an explicit lock/semaphore around each SDK's refetch, each proven by a
counting-mock burst test.
</objective>

## Artifacts this phase produces

- Go: `sync.Mutex` field + double-checked refresh in `internal/jwks/verifier.go`
- Java: `ReentrantLock` (or `synchronized`) around the refetch path in `JwksVerifier.java`
- C#: `SemaphoreSlim(1,1)` field in `JwksVerifier.cs` (new synchronization where none existed)
- Extended burst tests in each SDK's existing JWKS test file
