<objective>
Deliver the admin email-config REST API (FUNC-03 / D-13): six scope-nested singleton handlers
(GET/PUT/DELETE for `/api/v1/organizations/{org_id}/email-config` and `/api/v1/tenants/{tenant_id}/email-config`),

handlers and their DTOs (FUNC-01 / D-12), so generated SDKs model the OIDC start→callback contract.

Purpose: This is the real build of FUNC-03's admin surface plus the FUNC-01 documentation gap. It compiles
against 28-01's foundation (secret hygiene, delete_org_config, D-02 input semantics).
Output: A complete, RBAC-gated, secret-safe email-config CRUD surface + accurate public-SSO OpenAPI docs.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → email-config endpoint | Untrusted org_id/tenant_id path param vs the caller's JWT claim (IDOR surface) |
| stored config → GET response | Secret ciphertext must never round-trip to the client |
| new route → OpenAPI/permission map | A route missing from any leg of the triangle is a silent auth/doc gap |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-28-01 | Elevation of Privilege | Cross-tenant/cross-org IDOR on email-config endpoints | high | mitigate | Task 1: explicit `path_id != user.{org_id,tenant_id}` check before any repo call (settings.rs precedent); Task 3 asserts 403 |
| T-28-02b | Information Disclosure | Secret leaked in GET response body | high | mitigate | 28-01 skip_serializing + secret-omitting response DTO; Task 3 asserts body omits password/api_key |
| T-28-14 | Elevation of Privilege | Missing RBAC gate because route absent from ROUTE_PERMISSION_MAP | high | mitigate | Task 2 completes the triangle; route_openapi_parity_test enforces it (Pitfall 1) |
| T-28-15 | Spoofing | SSRF/egress via live credential validation at write time | low | accept | D-15: structural validation only, no outbound connectivity test — no SSRF surface introduced |
