<objective>
QUAL-03 — Error taxonomy correctness. Route genuine unique/index violations on the
mainstream create paths to HTTP 409 (AlreadyExists) instead of 500 (Migration), make
OAuth2 client-lookups distinguish a DB outage from invalid_client, and stop parse_uuid
mislabeling a corrupt read as "Migration failed". This is one of the two intentional
observable-behavior changes in Phase 29 (D-04): update any test asserting the old 500/
invalid_client behavior and add lock-in tests in the SAME commit that changes behavior.

Purpose: Correct HTTP semantics on create-path errors and remove an OAuth2 error-oracle
(a DB outage silently masquerading as bad credentials). Security-adjacent — lands first
(D-19) so QUAL-01's later AppState pass wires in already-final repo method bodies.

Output: A centralized helpers::classify_write_error, a new DbError::Serialization variant,
routed create/RELATE call sites, fixed OAuth2 client-lookup mapping, and new lock-in tests.
</objective>
