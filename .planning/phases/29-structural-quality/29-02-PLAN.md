<objective>
QUAL-04 — Transactional multi-statement mutations. Make role/permission edge deletes and
resource::delete tenant-predicated and atomic (closing the CQ-B07/SEC-058 cross-tenant
edge-strip family and the CQ-B46 child-guard TOCTOU), and make GDPR deletion setup a single
transaction so a mid-setup failure cannot strand an uncancellable purge (CQ-B39 residual).
This is the second of the two intentional observable-behavior changes in Phase 29 (D-04):
add lock-in tests and update any test asserting the old non-transactional behavior in the
SAME commit.

Purpose: Access-control (ASVS V4) tenant-isolation hardening — every mutating statement
independently re-asserts the tenant boundary; capture-then-mutate is atomic. Security-adjacent,
lands before AppState (D-19) so QUAL-01 wires in already-final repo bodies.

Output: Tenant-predicated inline transactions in role.rs/resource.rs, a new transactional
create_with_pending_flag, an updated gdpr.rs handler, and three lock-in tests.
</objective>
