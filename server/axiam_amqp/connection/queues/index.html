Well-known queue names used by AXIAM.

Constants§

AUDIT_EVENTS
Inbound audit events from external services.
AUDIT_EVENTS_DLQ
Dead-letter queue for AUDIT_EVENTS poison messages (CQ-B05).
AUTHZ_REQUEST
Inbound async authorization check requests.
AUTHZ_REQUEST_DLQ
Dead-letter queue for AUTHZ_REQUEST poison messages (CQ-B05).
AUTHZ_RESPONSE
Outbound authorization decision responses.
MAIL_OUTBOUND
Outbound async mail delivery queue (D-14).
MAIL_OUTBOUND_DLQ
Dead-letter queue for MAIL_OUTBOUND exhausted-retry messages (D-14).
NOTIFICATIONS
Outbound real-time event notifications.
WEBHOOK
Primary webhook delivery queue (CORR-03/D-07).
WEBHOOK_DLQ
Terminal dead-letter queue for exhausted webhook deliveries (CORR-03/D-07). Messages here are real and replayable — not silently dropped (see the DLX-wiring note on WEBHOOK/WEBHOOK_RETRY).
WEBHOOK_RETRY
Webhook retry-delay queue (CORR-03/D-07). No consumer is ever attached to this queue — a message published here with a per-message TTL (x-message-ttl/BasicProperties::with_expiration) dead-letters back to WEBHOOK via the default exchange once the TTL expires, giving RabbitMQ-native delayed retry without an in-process sleep tying up a consumer slot (D-07/Pitfall 5).
