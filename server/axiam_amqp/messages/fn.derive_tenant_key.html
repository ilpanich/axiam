pub fn derive_tenant_key(

Expand description

Derive a per-tenant AMQP signing subkey from the shared master key (SECHRD-08 / D-05a/b, T-25-19).

Uses HKDF-SHA256 (hkdf::Hkdf<Sha256>) — never hand-rolled concatenation hashing. The info parameter is domain-separated (fixed tag) and versioned (key_version) then tenant-scoped (tenant_id), so:

  • A signature produced with tenant A’s subkey never verifies under tenant B’s subkey, even though both derive from the same master key.
  • Bumping key_version (master key rotation) yields an entirely different subkey for the same tenant, without breaking in-flight messages signed under the prior version (D-05b).
