Admin email-config endpoints for organizations and tenants (FUNC-03 / D-13).

Six scope-nested singleton handlers (GET/PUT/DELETE for org and tenant scopes), gated by email_config:read (GET) and email_config:write (PUT/DELETE) per D-03, each enforcing an org/tenant ownership check before touching the repository (T-28-01 IDOR mitigation).

GET always returns the raw own-scope row (get_org_config / get_tenant_override) — never the merged get_effective_config (D-14). Secrets (SmtpConfig.password / ApiProviderConfig.api_key) never appear in any response body: both EmailConfig and EmailConfigOverride carry #[serde(skip_serializing)] secrets end-to-end (D-01, 28-01).

PUT performs structural validation only (D-15) — no live SMTP/API connectivity check is ever made at write time.
