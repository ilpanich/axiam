Configuration for the authentication service.

Fields§

§jwt_private_key_pem: String

PEM-encoded Ed25519 private key for JWT signing.

§jwt_public_key_pem: String

PEM-encoded Ed25519 public key for JWT verification.

§access_token_lifetime_secs: u64

Access token lifetime in seconds (default: 900 = 15 minutes).

§refresh_token_lifetime_secs: u64

Refresh token lifetime in seconds (default: 2_592_000 = 30 days).

§auth_code_lifetime_secs: u64

Authorization code lifetime in seconds (default: 600 = 10 minutes).

§jwt_issuer: String

JWT issuer (iss claim).

§oauth2_issuer_url: String

OIDC issuer base URL (e.g. “https://auth.example.com”). Used for OIDC discovery endpoint URLs. Falls back to jwt_issuer if unset.

§pepper: Option<SecretString>

Optional pepper prepended to passwords before Argon2id verification. Wrapped in SecretString so Debug/logging never leaks it by accident (SECHRD-12) — exposed only via .expose_secret() at the &str boundary where a pepper value is consumed.

§min_password_length: usize

Minimum password length for policy enforcement.

§mfa_encryption_key: Option<[u8; 32]>

256-bit AES-GCM key for encrypting TOTP secrets at rest. None disables MFA enrollment. Set programmatically (not from config files).

§federation_encryption_key: Option<[u8; 32]>

256-bit AES-GCM key for encrypting federation client secrets at rest. None means federation config create/update will fail at runtime. Set programmatically from AXIAM__AUTH__FEDERATION_ENCRYPTION_KEY (not from config files). Federation is optional — absence is warned, not fatal.

§allow_missing_aud_as_user: bool

When true, access tokens decoded without an aud claim are treated as axiam:user. Enables a back-compat window during the Phase 4 rollout while pre-Phase-4 tokens are still circulating. Default: true.

§cookie_secure: bool

When true (default), all auth cookies are marked Secure and are therefore sent only over HTTPS. Set AXIAM__AUTH__COOKIE_SECURE=false only in local HTTP development (e.g. http://localhost) — never in production or staging (D-18).

§mfa_challenge_lifetime_secs: u64

MFA challenge token lifetime in seconds (default: 300 = 5 minutes).

§totp_issuer: String

Issuer name shown in authenticator apps.

§max_failed_login_attempts: u32

Max consecutive failed login attempts before lockout (default: 5).

§lockout_duration_secs: u64

Initial lockout duration in seconds (default: 900 = 15 min).

§lockout_backoff_multiplier: f64

Exponential backoff multiplier for repeated lockouts (default: 2.0).

§max_lockout_duration_secs: u64

Maximum lockout duration in seconds (default: 3600 = 1 hour).

§email_verification_grace_period_hours: u32

Grace period in hours during which PendingVerification users can still log in (default: 24). Set to 0 to disable.

§password_reset_token_expiry_hours: u32

Password reset token expiry in hours (default: 1).

§webauthn_rp_id: String

WebAuthn Relying Party ID (typically the domain name, e.g. “auth.example.com”).

§webauthn_rp_origin: String

WebAuthn Relying Party origin (e.g. “https://auth.example.com”).

§webauthn_rp_name: String

WebAuthn Relying Party display name.

§jwt_encoding_key: Option<Arc<EncodingKey>>

CQ-B14: Pre-parsed Ed25519 signing key. Populated once at startup via resolve_keys(). When Some, token-issue functions skip PEM re-parsing. When None, they fall back to parsing from jwt_private_key_pem.

§jwt_decoding_key: Option<Arc<DecodingKey>>

CQ-B14: Pre-parsed Ed25519 verification key. Populated once at startup via resolve_keys(). When Some, token-verify functions skip PEM re-parsing. When None, they fall back to parsing from jwt_public_key_pem.

§hibp_breaker_threshold: u32

PERF-01: Consecutive HIBP failures/timeouts before the process-wide HibpBreaker trips open (default: 5). Overridable via AXIAM__AUTH__HIBP_BREAKER_THRESHOLD.

§hibp_breaker_cooldown_secs: u64

PERF-01: Cooldown in seconds the HibpBreaker stays open before allowing a half-open probe request (default: 30). Overridable via AXIAM__AUTH__HIBP_BREAKER_COOLDOWN_SECS.

pub fn effective_issuer(&self) -> &str

Effective issuer for JWT iss claims and OIDC discovery.

Returns oauth2_issuer_url when set, falling back to jwt_issuer. Trailing slashes are stripped so that the OIDC discovery issuer exactly matches token iss claims (OIDC Core §2 requires an exact string match).

Source

pub fn resolve_keys(&mut self) -> Result<(), String>

CQ-B14: Parse Ed25519 keys from PEM once and cache in Arc.

Call this once at startup after loading config from environment. After this returns Ok(()), all token functions skip per-call PEM parsing and use the cached keys instead.
