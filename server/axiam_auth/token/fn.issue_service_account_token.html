Issue a JWT access token for a service account authenticated via mTLS client certificate (device-auth cert-auth path, resolving TODO(T15)).

The sub claim is the service account’s user_id (mirrors the shape the device-auth handler previously passed to issue_access_token). jti is caller-supplied — service-account/device auth has no session row, so callers pass a random UUID. aud and scope mirror the values the device-auth call previously passed to issue_access_token (AUD_USER, no scopes) — only sub_kind differs (D-09). This claim is informational only (D-10): validation and authz are unaffected.
