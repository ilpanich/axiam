Mint a one-time first-run bootstrap setup token if the database has never been bootstrapped (SECHRD-04 / D-03b).

§Behavior

  • On a fresh (never-bootstrapped) database — no bootstrap_setup_token row yet AND no user row anywhere — a cryptographically random token is generated, its sha256 hash is persisted as the bootstrap_setup_token record ID, and the returned Some(token) carries the PLAINTEXT token so the caller can log it exactly once (the one deliberate secret-log exception, per D-03b). Only the HASH is ever persisted to the database.
  • On any subsequent boot — a setup token already exists, or at least one user already exists — this is a no-op and returns Ok(None); nothing is minted or re-logged.
