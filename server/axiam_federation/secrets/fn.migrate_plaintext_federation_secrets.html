Encrypt any remaining plaintext client_secret rows in federation_config.

Called once at server startup after schema migrations (D-12). The predicate client_secret_ciphertext IS NONE AND client_secret IS NOT NONE AND client_secret != "" is self-clearing: once a row is migrated it no longer matches the query, making repeated calls safe.

Per-row errors are logged and skipped — the loop continues to the next row. The failed rows will be retried on the next boot.

Returns the number of rows successfully migrated.
